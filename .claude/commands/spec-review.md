実装と仕様を照合し、乖離があれば修正する。

## 引数の解釈

- **機能名** (例: `/spec-review hover-preview`) → `docs/features/hover-preview.md` を参照仕様とする
- **タスク番号** (例: `/spec-review 001`) → `tasks/` 以下の `TASK-001-*.md` を探し、その「参照仕様」セクションに書かれたファイル群を参照仕様とする
- **引数なし** → 「何をレビューしますか?」と聞く

## 手順

1. 引数を解釈して参照仕様を決定する
2. 参照仕様ファイルを読む
3. 関連する実装ファイルを読む:
   - `src/lib/remark/` の remark プラグイン
   - `src/lib/build/` のインデックス生成ロジック
   - `src/components/` のコンポーネント
   - `src/pages/` の Astro ページ
4. 仕様と実装の差異を列挙して報告する:
   - 仕様にあるが未実装のもの
   - 実装にあるが仕様にないもの
   - 挙動が仕様と異なるもの
5. ユーザーに確認し、どちらを正とするかを決めて修正する:
   - 仕様が正しければ実装を修正する
   - 実装が正しければ仕様を更新する
6. タスクファイルがあればチェックリストを更新する

## 注意

- 小さな差異 (変数名・コメントなど) は無視してよい
- `src/lib/` が Astro/React を import していたらアーキテクチャ違反として報告する
